
Nombre del proyecto: Bookeando

Objetivos, descripción y funcionalidad del proyecto:

	- Búsquedas de libros.
	- Creación de una librería propia del usuario pudiendo eliminar libros y  añadirlos.
	- Aparatado de sugerencias.
	- Recomendar la página vía twitter.

URL del repositorio GitHub: https://github.com/FranHuzon/Proyecto_LM

URL de la página web: https://www.bookeando.herokuapp.com

¿Cuántas API web vas a usar en el proyecto?:

	-  API Google Books   https://developers.google.com/books/?hl=es-419
	
	-  API Twitter        https://developer.twitter.com/en/docs/
	
	
	Lenguaje(s) de marcas que utiliza(n) la(s) API: JSON
	
¿Qué tipo de autentificación tiene(n) la(s) API utilizada(s)?:

	- La aplicación usa API KEY, OAUTH1 y OAUTH2 para diferentes acciones.
	
¿Qué métodos vas a usar en las llamada a la API?: 

	- GET y POST

¿Cuantas cosas hace la aplicación?, podemos pensar cuantos @route va a tener nuestro programa Flask: 

	- Mi app contiene al menos 9 @routes, sin contar las de OAUTH.

¿Vas a hacer una página web válida y con hoja de estilo?:

	-  Ya está validada.
